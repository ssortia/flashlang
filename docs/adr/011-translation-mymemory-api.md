# 011. Внешний сервис перевода: MyMemory API

**Статус:** Принято

**Дата:** 2026-04-05

## Контекст

Ридер текстов требует перевода многословных выделений (фраз, предложений) на русский язык. Одиночные слова переводятся из локального словаря (см. ADR-010), поэтому внешний API вызывается только для фраз из нескольких слов. Нужен сервис, который:
- Не требует платёжной карты или регистрации для базового использования
- Поддерживает пару `en → ru`
- Имеет разумные лимиты для небольшой нагрузки

## Рассмотренные варианты

- **Google Translate API** — высокое качество, но платный (от $20/1M символов), требует GCP-аккаунта. Неоправданно для MVP.

- **DeepL API** — отличное качество для европейских языков. Бесплатный тариф: 500 000 символов/месяц. Требует регистрации и получения API-ключа. Хороший запасной вариант.

- **MyMemory API** — бесплатный (500 запросов/день без ключа, 1000/день с указанием email), не требует регистрации, поддерживает `en|ru`. Качество перевода приемлемо для учебного контекста.

- **LibreTranslate (self-hosted)** — опенсорсное решение, можно добавить в docker-compose. Требует значительных ресурсов (~2 GB RAM) и загрузки языковых моделей. Избыточно для старта.

## Решение

Выбран **MyMemory API** для MVP. Поскольку одиночные слова берутся из локального словаря (ADR-010), реальная нагрузка на API — только фразы, что существенно снижает количество запросов.

Формат запроса:
```
GET https://api.mymemory.translated.net/get?q={text}&langpair=en|ru
```

В NestJS создаётся `TranslationModule` с `TranslationService`, который:
1. Делает HTTP-запрос к MyMemory через встроенный `fetch` (Node 22+)
2. Оборачивает ответ в типизированный DTO
3. Логирует ошибки через `nestjs-pino`, но не пробрасывает детали MyMemory наружу

Фронтенд обращается к `POST /translation` (собственный API), а не напрямую к MyMemory — это изолирует внешнюю зависимость и позволит сменить провайдера без изменений на клиенте.

## Последствия

### Положительные
- Нулевые затраты на старте, работает без настройки
- Единая точка замены провайдера (только `TranslationService`)
- Скрытие ключей и деталей интеграции от клиента
- Сниженная нагрузка на API благодаря локальному словарю для одиночных слов

### Отрицательные / компромиссы
- 500 запросов/день без ключа — при активном использовании фраз может не хватить
- Качество перевода ниже, чем у Google Translate или DeepL
- Зависимость от внешнего сервиса без SLA

### Риски
- MyMemory может изменить условия или стать недоступным. Миграция на DeepL: заменить один метод в `TranslationService`, добавить `DEEPL_API_KEY` в `.env`.
