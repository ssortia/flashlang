# ADR-013: Дизайн-система Flashlang

**Статус:** Принято  
**Дата:** 2026-04-10

## Контекст

Проект использовал дефолтную shadcn/ui тему (тёмно-синий primary, серые акценты) без собственного характера. Отсутствовали:

- semantic-цвета (success, warning, info)
- токены для уровней знания (0–5), используемых в тренировке и словаре
- пользовательский шрифт
- боковая навигация (только горизонтальная в header)

## Решение

### Цветовая палитра

**Primary — Indigo** (`hsl(239 84% 67%)`): ассоциируется с интеллектом, доверием, обучением.  
**Accent — Violet** (`hsl(262 83% 96%)`): мягкий, для выделения активных элементов.  
**Success — Emerald**: правильные ответы, освоенные слова.  
**Warning — Amber**: промежуточные уровни.  
**Destructive — Rose**: неверные ответы.

### Типографика

**Geist Sans** через `next/font/google` — шрифт от Vercel, хорошо оптимизирован для Next.js, нейтральный и читаемый.

### Токены уровней знания

6 CSS-переменных `--knowledge-0..5`: серый → amber → yellow → lime → green → emerald.  
Используются в компоненте `KnowledgeBadge`, который применяется в `CardScreen` и `VocabularyTable`.

### Навигация

Заменена горизонтальная nav в header на **sidebar**:

- Desktop: фиксированный `w-64` слева
- Mobile: CSS-overlay drawer (без Radix — нет shadcn CLI для добавления Sheet)
- Управление состоянием через `AppShell` Client Component (паттерн: передача Server Components как `children`-пропсов в Client Component)

## Последствия

- Все shadcn/ui компоненты автоматически получают новую палитру через CSS-переменные
- `KnowledgeBadge` — единственное место определения цветов и меток уровней знания
- `AppShell` оборачивает весь dashboard — любые изменения layout делаются здесь
- При добавлении shadcn-компонентов через `shadcn add` нужно вручную настраивать `components.json` (CLI не настроен)
