# 012. Модель прогресса изучения слова: knowledgeLevel 0–5

**Статус:** Принято

**Дата:** 2026-04-05

## Контекст

Пользователь добавляет слова в словарь и хочет отслеживать, насколько хорошо он знает каждое слово. Нужна простая числовая метрика, которая:
- Отражает субъективную оценку пользователя
- Используется для приоритизации слов на тренировке
- Визуально подсвечивает слова в ридере

## Рассмотренные варианты

- **Вариант A: бинарное состояние (известно / неизвестно)** — просто, но не даёт градации. Нельзя отличить «видел один раз» от «знаю уверенно».

- **Вариант B: 5-ступенчатая шкала SM-2 (интервальные повторения)** — полноценный алгоритм с датой следующего повторения, фактором лёгкости и историей ответов. Высокое качество, но значительная сложность реализации на старте.

- **Вариант C: целое число 0–5 с ручной/автоматической коррекцией** — простая шкала, понятная пользователю, достаточная для приоритизации тренировок. Уровень обновляется вручную при добавлении и автоматически по результату тренировки.

## Решение

Выбран **Вариант C**: поле `knowledgeLevel: Int` в модели `UserWord`, значения от 0 до 5.

### Семантика уровней

| Уровень | Значение  | Описание                             |
| ------- | --------- | ------------------------------------ |
| 0       | Незнакомо | Слово новое, встречается впервые     |
| 1       | Видел     | Встречал, но перевод не помню        |
| 2       | Сложно    | Знаю с трудом, нужна подсказка       |
| 3       | Средне    | Вспоминаю с усилием                  |
| 4       | Легко     | Знаю уверенно, редко ошибаюсь        |
| 5       | Освоено   | Мгновенное узнавание, ошибки редки   |

### Обновление уровня

При добавлении слова из ридера: пользователь выбирает уровень вручную (0–5).

При тренировке (флеш-карточки):
- Ответ «знал» → `knowledgeLevel = min(5, current + 1)`
- Ответ «не знал» → `knowledgeLevel = max(0, current - 1)`

### Приоритизация тренировки

Слова для тренировки отбираются в порядке возрастания `knowledgeLevel` (сначала менее известные), с ограничением по количеству (по умолчанию 20 слов за сессию).

### Визуализация в ридере

Слова из словаря подсвечиваются цветом по уровню знания:
- 0–1: красный (незнакомо)
- 2–3: жёлтый (учится)
- 4–5: зелёный (освоено)

## Последствия

### Положительные
- Простая реализация: одно числовое поле, без дополнительных таблиц
- Понятная пользователю шкала
- Достаточно для базовой приоритизации тренировок
- Легко расширить до полноценного SM-2 позже (добавить поля `easeFactor`, `interval`, `nextReviewAt`)

### Отрицательные / компромиссы
- Не учитывает время между повторениями (не настоящие интервальные повторения)
- Приоритизация упрощена: нет понятия «просроченности» карточки

### Риски
- При большом словаре (500+ слов) простая сортировка по уровню не обеспечивает оптимальный порядок повторений. Решение: добавить `lastReviewedAt` в критерии выборки или перейти на SM-2 в следующей версии.
